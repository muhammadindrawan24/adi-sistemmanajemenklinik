# Fitur Pesan Antrian Online H-1

## Penjelasan Lengkap dari Segala Arah

### Apa itu?

Pesan Antrian Online H-1 memungkinkan pasien **pesan antrian sehari sebelumnya** (misal pesan malam ini untuk besok). Berbeda dengan fitur sekarang yang hanya bisa ambil antrian hari yang sama.

**Contoh:** Pasien pesan jam 21:00 hari Senin → dapat nomor antrian untuk hari Selasa.

---

## 1. DATABASE

### Perubahan Tabel `queues`
```
+ visit_date DATE NOT NULL DEFAULT CURRENT_DATE
```
- Field baru: `visit_date` — tanggal pasien akan diperiksa
- `created_at` tetap ada (kapan pesan dibuat)
- Nomor antrian di-reset per `visit_date`, bukan per `created_at`

### Perubahan Fungsi `create_queue()`
- Nomor antrian dihitung berdasarkan `visit_date`, bukan hari ini
- Filter: `WHERE queue_number LIKE 'U%' AND visit_date = p_visit_date`
- Pasien pesan untuk besok → dapat U001 besok, bukan U716 hari ini

### Perubahan Trigger `handle_queue_selesai()`
- Auto-call next hanya untuk antrian dengan `visit_date = CURRENT_DATE`
- Jangan panggil antrian yang dijadwalkan untuk besok

---

## 2. UI PASIEN (take-queue/page.tsx)

### Yang berubah
- **Tambah date picker** — pilih tanggal (hanya besok yang bisa dipilih)
- **Jadwal dokter** — fetch berdasarkan tanggal yang dipilih, bukan hari ini
- **Jam operasional** — hanya berlaku untuk booking hari ini, booking besok bebas jam
- **Cek antrian aktif** — filter berdasarkan `visit_date` juga

### Flow baru
```
1. Pilih tanggal (Hari ini / Besok)
2. Pilih Poli
3. Lihat jadwal dokter (berdasarkan tanggal)
4. Isi keluhan
5. Klik "Pesan Antrian"
6. Dapat nomor + info tanggal kunjungan
```

---

## 3. UI "ANTRIAN SAYA" (my-queue/page.tsx)

### Yang berubah
- **Tampilkan tanggal kunjungan** di kartu antrian
- **Filter by visit_date** saat fetch data
- **Tampilkan semua antrian aktif** (hari ini + besok)

## 5. MONITOR (monitor/page.tsx)

### Yang berubah
- **Filter by `visit_date = CURRENT_DATE`** — tampilkan hanya antrian hari ini
- Tidak berubah banyak, monitor tetap fokus hari ini

---

## 6. FLOW LENGKAP

### Booking Hari Ini (sama seperti sekarang)
```
Pasien buka halaman → Ambil Antrian → Pilih "Hari Ini" → Pilih Poli → Dapat nomor
```

### Booking H-1
```
Pasien buka halaman → Ambil Antrian → Pilih "Besok" → Pilih Poli → Lihat jadwal → Dapat nomor untuk besok
```

### Hari Besok Tiba
```
Petugas buka panel antrian → Tab "Hari Ini" → Lihat semua antrian (termasuk yang dipesan H-1)
→ Panggil pasien seperti biasa
```

---

## 7. BATASAN / ATURAN

| Aturan | Keterangan |
|--------|-----------|
| Maksimal H-1 | Hanya bisa pesan untuk besok, bukan lusa |
| Satu antrian aktif | Pasien tidak boleh punya 2 antrian dengan status aktif |
| Nomor urut per hari | U001 hari ini terpisah dari U001 besok |
| Batalkan | Pasien bisa batalkan antrian H-1 sebelum hari kunjungan |
| Operator hours | Booking besok tidak dibatasi jam operasional |
